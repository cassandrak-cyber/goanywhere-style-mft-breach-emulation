Lessons Learned

Reflection

This project helped me understand that threat emulation is much more than simply demonstrating an attack. Before building this lab, I mainly thought about the offensive side of cybersecurity—how an attacker gains access and steals data. As I worked through each phase, I realized that the real value comes from understanding what evidence is left behind and how defenders can use that evidence to detect and investigate malicious activity.

One of the biggest lessons I learned was the importance of documentation. Taking screenshots, mapping each step to the MITRE ATT&CK framework, preserving server logs, and generating a SHA-256 hash showed me how security teams build a complete picture of an incident rather than focusing on a single event.

This project also reinforced how valuable realistic lab environments can be. Building the simulated managed file transfer server and working through the attack step by step helped me connect concepts I learned in class to a practical scenario. Instead of only reading about attacker behavior, I was able to observe how actions such as file discovery, collection, and staging generated artifacts that could later be investigated.

Finally, this project strengthened my understanding of the relationship between offensive and defensive security. Every action performed by the simulated attacker created opportunities for detection. That perspective has changed the way I approach cybersecurity. Instead of asking only how an attack works, I now think about how I would detect it, investigate it, and help prevent it from happening again.
